## 核心逻辑说明

### Bug #1 修复：三层缓存优先守卫

```
Layer 1 (Service): _tryFreshCache → 新鲜缓存直接返回 Ok → 不发 HTTP
Layer 2 (Provider): 已有 zdbkEverythingProvider 内存缓存 + 空数据守卫
Layer 3 (BackgroundRefresher): db.getFreshCachedWebPage != null → 跳过拉取
```

**Flow**: Dashboard 打开 → Provider watch（不 invalidate）→ Service `_tryFreshCache` 命中 → 直接返回缓存数据 → UI 即时渲染

### Bug #2 修复：时间戳一致性

- `DataStatusManager.refreshFreshness`: cacheKey=null 源不再伪造 `lastFetchedAt`，初始"从未"
- `QuickConnectScreen._freshnessCard`: subtitle 统一用 `relativeTime`，消除 badge/subtitle 矛盾
- `BackgroundRefresher._silentRefresh`: 拉取完成后 `invalidate(dataStatusManagerProvider)` 同步面板时间戳
- 补充注册 ZDBK 主修成绩 + 实践分数

## 潜在影响

- **低风险**: 缓存优先守卫在 `_withAutoRelogin` 之前执行，仅新增快速路径；网络失败后过期缓存兜底逻辑保持不变
- **Dashboard**: 首次打开不再自动拉数据 → 用户可能看到较旧缓存 → 手动 F5 或数据状态面板刷新即可
- **BackgroundRefresher**: 跳过新鲜数据 → 后台流量大幅降低
- **getPracticeScores**: 缓存格式从 HTML 改为 JSON → 旧缓存自动失效（解析失败走网络），新缓存正常
