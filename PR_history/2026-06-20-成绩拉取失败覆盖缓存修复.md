# PR_history/2026-06-20-成绩拉取失败覆盖缓存修复.md

## 修改目的

修复 Bug：ZDBK 成绩拉取失败时，`getEverything()` 静默将错误折叠为空数据并返回 `Ok`，导致内存缓存被空数据覆盖。用户看到全零成绩 + 旧新鲜度时间戳。

**反馈来源**: `test/feedback/20260619_171734_root_🐛_Bug/feedback.md`

- `lib/features/zdbk/providers/zdbk_provider.dart` — `zdbkEverythingProvider` 空数据守卫
- `agent_contributing/experiences/2026-06-20-geteverything-silent-error-swallowing.md` — 经验卡片

## 核心逻辑说明

**根因**: `getEverything()` 总是返回 `Ok`，即使 `getTranscript()` 和 `getExams()` 都失败。错误被 `.fold((g) => g, (_) => [])` 折叠为空列表，provider 的 `Err` 回退分支永远不触发。

**三处改动**:

1. **`getEverything()`** — 双方都失败时返回 `Err`（不再折叠为空 `Ok`）。部分失败（transcript 失败但 exams 成功）仍返回 `Ok` 但记录警告。

2. **`zdbkEverythingProvider`** — 即使 `getEverything` 返回 `Ok`，若 `grades.isEmpty && exams.isEmpty` 且内存缓存非空，保留缓存不覆盖。

3. **`getTranscript()`** — 将缓存回退从泛型 `_withAutoRelogin`（返回 `List<Map> as List<Grade>`，类型不安全）移到方法自身，正确执行 `Grade.fromJson` 反序列化。

**防御层级**:
```
网络失败 → getTranscript Err → getEverything Err → provider 缓存回退 ✅
网络"成功"空数据 → getEverything Ok(空) → provider 空检查 → 保留缓存 ✅
```

## 潜在影响

- **低风险**: 仅改变错误传播路径，正常成功路径不受影响。
- `getTranscript` 的文件缓存回退现在返回正确类型的 `List<Grade>`（之前是 `List<Map>` 强转），修复了一个隐式的类型安全隐患。
- 其他使用 `_withAutoRelogin` + `fallbackKey` 的方法（`getTimetable`、`getCourseOfferings` 等）仍有相同的泛型缓存回退类型不匹配，但未在此次修复范围内（见经验卡片注意事项）。
