# Bug 反馈规则

> 感谢你花时间反馈问题！请严格按照以下模板填写，信息不完整的反馈可能会被忽略或关闭。

---

## 模板

```markdown
### 问题描述
（清晰简洁地描述问题现象）

### 严重程度
- [ ] 致命（崩溃、数据丢失、主要功能完全不可用）
- [ ] 严重（主要功能异常但可绕过，或核心体验严重受损）
- [ ] 一般（次要功能异常，有临时替代方案）
- [ ] 轻微（UI 错位、文案错误、不影响使用的瑕疵）

### 复现步骤
1. 打开 App 并登录
2. 进入「XXX」页面
3. 点击「YYY」按钮
4. 观察到错误现象

### 预期行为
（描述正常情况下应该发生什么）

### 实际行为
（描述实际发生了什么，可附截图/录屏）

### 环境信息
- 设备型号：例如 Pixel 6 / iPhone 13
- 操作系统版本：例如 Android 14 / iOS 17.2
- App 版本号：（可在设置/关于中查看）
- 网络环境：Wi-Fi / 5G / 校园网（如有特殊代理/RVPN请注明）

### 附加信息
- 是否稳定复现：是 / 否（如非稳定，复现概率约 %）
- 相关日志：（可粘贴 `Log` 输出的关键片段）
- 其他备注：
```

---

## 提交前检查清单

- [ ] 我已搜索过现有 Issues，确认没有重复报告
- [ ] 我已填写所有必填字段（问题描述、严重程度、复现步骤、环境信息）
- [ ] 我已附上必要的截图或日志（如适用）
- [ ] 我确认使用最新版本后问题依然存在（如无法确认请注明）

---

## 特殊说明

- **关于安全漏洞**：请勿在公开 Issue 中描述涉及 API Key、Cookie、Token 等敏感信息，应发送邮件至 `security@example.com`（替换为实际安全邮箱）。
- **关于网络相关 Bug**：请额外说明是否开启了代理/RVPN，以及校园网环境是否特殊。
- **关于登录认证问题**：请注明涉及的服务（ZDBK / Courses / PTA 等），不要笼统说“登录失败”。

---

> 未按模板填写的 Issue 将被标记为 `needs-info`，3 天内无更新则自动关闭。
