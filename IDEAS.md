# 新想法反馈规则

> 感谢你为项目贡献创意！一个好的想法不仅需要“做什么”，更需要“为什么做”和“怎么做”。请严格按照以下模板填写，信息越完整，越有可能被采纳。

---

## 模板

```markdown
### 想法标题
（一句话概括你的建议）

### 解决的问题
（描述当前存在的痛点、缺失的功能，或用户未被满足的需求）

### 建议的解决方案
（详细描述你希望实现的功能或改进，可附草图、流程图或参考示例）

### 预期收益
- [ ] 提升效率（具体说明节省多少时间/操作步骤）
- [ ] 提升稳定性/安全性
- [ ] 改善用户体验（如减少困惑、更直观）
- [ ] 新场景支持（具体说明）
- [ ] 其他：

### 可能的实现思路（可选）
（如果你有技术背景，可以给出大致的实现方向，例如：复用现有 XXX 模块，接入 YYY API，采用 ZZZ 方案）

### 涉及的服务/平台（多选）
- [ ] ZDBK 教务网
- [ ] Courses 学在浙大
- [ ] Classroom 智云课堂
- [ ] PTA
- [ ] DeepSeek AI
- [ ] 一卡通（BlueWare）
- [ ] 本地数据/存储
- [ ] UI/交互
- [ ] 其他：

### 优先级建议
- [ ] 紧急（不解决会影响核心使用）
- [ ] 高（能显著改善大部分用户的体验）
- [ ] 中（有实用性，但不是必须）
- [ ] 低（锦上添花，未来考虑）

### 附加信息
- 是否愿意参与测试该功能：是 / 否
- 是否愿意提供更多细节/讨论：是 / 否
- 参考资料/链接：
```

---

## 提交前检查清单

- [ ] 我已搜索过现有 Issues，确认没有重复建议
- [ ] 我已填写所有必填字段（标题、问题描述、解决方案、预期收益）
- [ ] 我已合理评估该功能的复杂度和影响范围
- [ ] 我理解采纳与否取决于项目方向、维护成本和现有优先级

---

## 特殊说明

- **关于第三方服务依赖**：如果建议涉及新的外部 API（如新教务系统、新 AI 服务），请提供该服务的公开文档链接或认证方式说明。
- **关于 UI/交互**：如有可能，附上手绘草图、Figma 链接或参考现有 App 的类似设计。
- **关于性能/安全**：如果建议涉及敏感数据或高频率请求，请特别说明潜在风险。

---

> 好的想法值得被看见！我们会定期审阅所有提案，并给出反馈（采纳、推迟或婉拒）。未被立即采纳不代表想法不好，可能只是时机或资源问题。
